Spring Security 研究(1)-架构

发表于 2018-08-20

        Spring Security和Apache Shiro是目前使用比较多的开源权限框架。比较来说，Spring Security是基于Spring framework体系的，和Spring框架整合方便。Apache Shiro框架比较灵活，自定义方便。总的来说，如果是需要深度定制的权限框架，两者都是需要进行定制开发的。

Spring Security vs Apache Shiro

Shiro相对Security来说，具有简单、灵活，定制方便。如果项目是Spring全家桶系列，则Spring Security也是不错的选择

认证的核心接口是 AuthenticationManager，它包含如下接口

一个AuthenticationManager,需要在authenticate()中实现下列行为之一:

  • 认证成功，返回Authentication(通常它的authenticated属性为true authenticated=true)
  • 认证失败，抛出AuthenticationException异常
  • 无法判断，返回null

AuthenticationManager主要实现为ProviderManager, 他代理一个AuthenticationProvider访问链。

Customizing Authentication Managers

Spring Security provides some configuration helpers to quickly get common authentication manager features set up in your application. The most commonly used helper is the AuthenticationManagerBuilder which is great for setting up in-memory, JDBC or LDAP user details, or for adding a custom UserDetailsService.
